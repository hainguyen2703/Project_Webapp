# Implementation Plan: Add Login and Home Button

**Branch**: `006-add-login-home-buttons` | **Date**: 2026-05-26 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/006-add-login-home-buttons/spec.md`

## Summary

Add session-based login/logout and a "Home" nav link. A new `login_service.py` module uses `werkzeug.security.check_password_hash` to verify credentials against the existing `UserAccount` model (status must be `"active"`). On success, `session["user_id"]` is set. `base.html` gains a persistent "Home" link and conditional Login/Log-out nav items driven by a `logged_in` boolean passed from every page-rendering route. No schema changes. No new Python packages.

## Technical Context

**Language/Version**: Python 3.13.2
**Primary Dependencies**: Flask 3.1.3, Flask-SQLAlchemy 3.x, Werkzeug (transitive — `check_password_hash`)
**Storage**: SQLite via Flask-SQLAlchemy (`UserAccount` read-only for login); Flask cookie session (`session["user_id"]`)
**Testing**: pytest 9.0.3
**Target Platform**: Local development server (Flask built-in)
**Project Type**: Flask server-rendered web application
**Performance Goals**: Login flow completes in under 60 seconds (SC-001 — trivially met by a single indexed DB lookup)
**Constraints**: No new Python packages; use existing Werkzeug `check_password_hash`; reuse Flask session mechanism
**Scale/Scope**: Single-developer app; <10 concurrent users; 48 existing tests baseline
